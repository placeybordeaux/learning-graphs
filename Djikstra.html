Dijkstra's Algorithm

A shortest path algorithm

Overview

Dijkstra's algorithm finds the shortest path between an initial vertex and a goal vertex. Dijkstra's algorithm assigns a tentative distance value to every vertex. It makes it zero for the initial vertex and infinity for the rest of the vertices. It also creates a visited set, which initially has just the starting node, and an unvisited set, which creates the rest of the nodes.

It starts running at the initial vertex. For the current vertex, add the vertex to the visited set and remove it from the unvisited set. Then add the tentative distance to the current vertex plus the weight of the edge between the current vertex and its unvisited neighbors. If this value is less than the calculated tentative distance for any one of those unvisited neighbors, replace the stored tentative distance with newly calculated value. If we just visted the goal state, we're done. Otherwise, go to the unvisited node with the lowest tentative distance.

Pseudo Code

Input: Graph G = (V,E), directed or undirected; positive edge lengths fle : e 2 Eg; vertex s 2 V
Output: For all vertices u reachable from s, dist(u) is set to the distance from s to u.

  1. for all u 2 V :
  2.     dist(u) = 1
  3.     prev(u) = nil
  4.     dist(s) = 0
  5.     while Q is not empty:
  6. H = makequeue(V) (using dist-values as keys)
  7. while H is not empty:
  8.     u = deletemin(H)
  9.     for all edges (u; v) 2 E:
  10.         if dist(v) > dist(u) + l(u; v):
  11.             dist(v) = dist(u) + l(u; v)
  12.             prev(v) = u
  13.             decreasekey(H; v)

Complexity

Time Complexity: O((|V| + |E|) log(|V|))
Space Complexity: O(|V|)
