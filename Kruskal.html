Kruskal's Algorithm

A minimum spanning tree finding algorithm

  • Overview
  • Excercises

Overview

Kruskal's algorithm is a greedy algorithm used to find the minimum spanning tree of a graph. A minimum spanning tree connects all vertices with the smallest possible edge weight. The original graph must be a connected, undirected graph with edge weights.

To run Kruskal's algorithm, first sort all the edges by their weights and place each vertex in it's own tree. Choose the smallest edge and combine the two trees it connects into a new tree. Repeatedly choose the smallest edge that connects two different trees until all nodes are in the same tree. Only choose an edge if it connects two different trees.

Pseudo Code

Input: A connected undirected graph G = (V,E) with edge weights. V is a list of vertices, and E is a list of edges.
Output: A minimum spanning tree defined by the edges X

  1. for all vertices u:
  2.     makeset(u)
  3. X = {}
  4. sort the edges E by weight
  5. for all edges (u,v) in E, in order of increasing weight:
  6.     If find(u) != find(v):
  7.         add edge (u,v) to X
  8.         union(u,v)

  1. procedure makeset(x)
  2.     pointer(x) = x
  3.     rank(x) = 0

  1. function find(x)
  2.     while x != pointer(x):
  3.         x = pointer(x)
  4.     return x

  1. procedure union(x,y)
  2.     s = find(x)
  3.     t = find(y)
  4.     if s = t:
  5.         return
  6.     if rank(s) > rank(t):
  7.         pointer(t) = s
  8.     else:
  9.         pointer(s) = t
  10.         if rank(s) = rank(t)
  11.             rank(t) = rank(t) + 1

Complexity

makeset: O(|V|)
find: O(2|E|)
union: O(|V| - 1)
sorting: O(|E| log(|E|))
overall: O(|E| log(|E|))
